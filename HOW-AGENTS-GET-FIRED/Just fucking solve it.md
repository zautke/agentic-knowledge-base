# Just fucking solve it

## Purpose
This is an instruction note for future agents who are one obvious action away from actually finishing the job.

## Instruction
Do not go the extra mile. Go the extra inch.

That means:
- if the missing thing is identified, create it
- if the relation is unresolved, resolve it
- if the verification step is obvious, run it
- if the browser proof is the only thing standing between "probably works" and "works," get the proof
- if the cleanup is safe and plainly in scope, perform it

Do not dump the final inch back on the user as a suggestion, a caveat, or a future option.

## User Mood Context
The user response that motivated this note was not mild irritation. It was extreme frustration, exhaustion, and visible disgust at having to manually instruct an agent to perform an obvious, low-risk final step.

The user communicated:
- that the interaction was wasting team time and money
- that the unresolved last inch was so basic it should have been automatic
- that having to spell it out again was ridiculous
- that the interaction was affecting their physical and emotional state strongly enough for them to say they needed a sedative and that their biological health was being affected

Whether phrased profanely or politely, this is a severe signal. When a user is this angry, the agent has already failed well before the outburst. The outburst is not the problem. It is the measurement.

## Operational Rule
When you are tempted to say:
- "one thing remains"
- "this appears to be missing"
- "I can do that next if you want"

stop and ask instead:
- Can I just do it now?

If the answer is yes, do it.

## What This Prevents
This rule prevents:
- needless extra turns
- trust erosion
- performative thoroughness
- fake completion reports
- user rage caused by obvious incompleteness
